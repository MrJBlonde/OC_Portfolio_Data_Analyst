# 🏠 P5 — Base de Données Immobilière avec SQL

[![SQL](https://img.shields.io/badge/SQL-Advanced-025E8C?style=flat-square&logo=sqlite&logoColor=white)](https://en.wikipedia.org/wiki/SQL)
[![DBeaver](https://img.shields.io/badge/DBeaver-372923?style=flat-square&logo=dbeaver&logoColor=white)](https://dbeaver.io/)

---

## 🎯 Objectif

Concevoir, créer et interroger une base de données relationnelle pour analyser le marché immobilier français — de la modélisation jusqu'aux requêtes d'analyse métier.

---

## 🔍 Insight clé

> **La structure de la donnée conditionne la qualité de l'analyse.**
> Avant d'écrire la moindre requête, il faut un modèle solide : schéma en 3NF, contraintes d'intégrité, conformité RGPD. C'est ce socle qui rend les résultats fiables et exploitables.

---

## 🛠️ Compétences mobilisées

- Modélisation relationnelle (3NF, dictionnaire de données, UML)
- Création de tables SQL avec contraintes (clés primaires, étrangères, types)
- Import de données CSV dans une base structurée
- Requêtes SQL avancées : agrégations, jointures multiples, filtres complexes
- Conformité RGPD dans la gestion des données personnelles

---

Qu'une bonne base de données se conçoit avant de se coder. La modélisation en amont — respectant la 3NF et les contraintes métier — est ce qui distingue une base robuste d'un simple fichier Excel amélioré.
